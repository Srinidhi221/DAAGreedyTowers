§ 01 Variable Definitions
N=board size (N × N grid)
E=number of empty cells (≤ N²)
b=branching factor (choices per node)
d=recursion depth of search
b'=effective b after MRV/LCV ordering
T(d)=time cost at depth d
§ 02 MRV — Minimum Remaining Values
Cell Scan Scanning all unassigned cells to find most-constrained variable
1.

Iterate over the entire N × N board to locate all unassigned cells. There are at most O(N²) cells to examine.

Scan all cells: O(N²)
Domain Size Computing remaining legal values per cell
1.

For each candidate cell, count how many values from 1..N are still legal. This requires checking the cell's row, column, and any structural constraints — each up to N peers with N possible values.

2.

Conflict check per (cell, value) pair: scan row peers + column peers = 2N cells → O(N) per value, O(N²) per cell.

Domain computation per cell: O(N) values × O(N) conflict checks = O(N²) Over all cells: O(N²) × O(N²) = O(N⁴)
📐
MRV Total Cost
O( N⁴ )
Selecting the most-constrained variable requires scanning N² cells and computing O(N²) domain size each.
§ 03 LCV — Least Constraining Value
Value Ordering Sorting candidate values by how many neighbours they constrain
1.

For each of the up to N candidate values at the chosen cell, count how many legal moves that value would eliminate from all neighbouring unassigned variables.

2.

Each neighbour impact check: up to O(N) neighbouring unassigned cells, each requiring O(N) constraint re-evaluation → O(N²) per candidate value.

3.

Repeated over N values, then combined with the MRV variable-selection pass over all N² cells:

LCV cost per variable: O(N) values × O(N²) impact = O(N³) Combined with MRV selection: O(N²) cells × O(N³) = O(N⁵)
🔢
MRV + LCV Combined Cost
O( N⁵ )
The heuristic overhead per decision node. This is dominated by the exponential search tree in the worst case.
§ 04 Backtracking Search — Recurrence Derivation
Recurrence Branching structure at each search node
1.

At each node in the search tree, the algorithm tries each of b legal values for the chosen cell and recurses to depth d − 1:

Recurrence: T(d) = b · T(d − 1) Base case: T(0) = O(1) ← board is fully assigned
Unrolling Expanding the recurrence to closed form
1.

Expand step by step:

T(d) = b · T(d − 1) = b · b · T(d − 2) = b² · T(d − 2) = b³ · T(d − 3) = ... = bᵈ · T(0) = bᵈ
2.

The search tree has bᵈ leaf nodes and O(bᵈ) total nodes.

Substitution Worst-case parameters for the N × N Towers puzzle
1.

Branching factor: each empty cell can hold any digit from 1 to N before constraint checks, so b = N in the worst case.

2.

Recursion depth: the algorithm must fill all E ≤ N² empty cells, so d = N² in the worst case.

b = N (up to N digit choices per empty cell) d = N² (up to N² empty cells in the grid) Substituting: T = b^d = N^(N²) ⟹ Worst-case search tree cost: O( N^(N²) )
⚠
Backtracking Search Tree · Worst Case
O( N^(N²) )
Super-exponential. For N = 6, this is ~1.5 × 10²⁸ nodes — computationally intractable without heavy pruning.
§ 05 Total Cost Breakdown
Aggregation Combining heuristic overhead with search tree size
1.

At each of the O(N^(N²)) nodes in the search tree, the strategy runs MRV + LCV to select the next variable and order values.

2.

The MRV + LCV overhead per node is O(N⁵). So the total cost is:

Heuristic overhead per node: O(N⁵) Search tree size: O(N^(N²)) ───────────────────────────────────────────────── Full cost: O(N⁵) × O(N^(N²)) = O(N^(N² + 5)) Since N² + 5 ≈ N² for large N: ⟹ Dominant term: O( N^(N²) )
3.

The heuristic polynomial factor N⁵ is completely absorbed by the super-exponential N^(N²). The asymptotic class is unchanged.

✓
Practical Effect of MRV + LCV
b' ≪ N (in practice)
MRV and LCV reduce the effective branching factor b′ significantly below N by pruning poor branches early. The worst-case class remains O(N^(N²)) but typical runtime is far lower.
