BT Forward Check vs Trap Setter

Formal worst-case derivation comparing StrategyBTForwardCheck (three-tier backtracking with forward checking) against the Trap Setter strategy (backtracking + MRV + LCV), applied to the Towers puzzle as a CSP.

BT + Forward Check MRV (implicit) 3-Tier Relaxation Trap Setter MRV + LCV Pure Backtracking
§ 01 Variable Definitions
N=board size (N × N grid)
E=number of empty cells (≤ N²)
b=branching factor (choices/node)
d=recursion depth
D=domain size after FC pruning
T=number of tiers attempted
§ 02 Strategy Overviews
Strategy A
BT Forward Check
O(N² · D^N²)
Three-tier backtracking with domain-based forward checking. Prunes branches the moment any unassigned cell loses all values. Falls back through tiers: Perfect → Local Safe → Desperation.
Strategy B
Trap Setter
O(N^(N²))
Backtracking augmented with MRV variable ordering and LCV value ordering. No domain propagation — relies on constraint checking only. Worst case explores the full exponential search tree.
§ 03 BT Forward Check — Derivation
OUTER LOOP findBestMove() — Scanning all (cell, value) pairs
1.

Outer double loop iterates over all cells: O(N²) cells.

2.

Inner loop tries each of up to N values per cell.

Outer scan cost: O(N²) cells × O(N) values = O(N³)
FC STEP applyForwardChecking() — Per candidate move
1.

For each placed value, propagate domain removals across the row and column: 2N neighbour cells checked.

2.

After each removal, isDomainEmpty() scans N values → O(N) per check.

FC cost per move: O(N) neighbours × O(N) domain scan = O(N²)
TIER 1 isPathSafe() — Full DFS with FC (MRV-ordered)
1.

isPathSafe() recursively selects the cell with minimum remaining domain (implicit MRV), branching over its domain values.

2.

At each recursive call, the FC check propagates and prunes branches with empty domains immediately. In the worst case all N² cells must be filled and each has domain size D after pruning:

Recurrence: T(d) = D · T(d − 1) per recursion level Unrolled: T = D^(N²) Per outer (cell, value) candidate: O(N²) · O(D^(N²)) Full Tier-1 cost: O(N³) · O(D^(N²)) = O(N³ · D^(N²))
3.

In practice, FC heavily prunes D. But worst case D → N, giving O(N³ · N^(N²)) ≡ O(N^(N²+3)).

TIER 2 / 3 Fallback and Desperation tiers
Tier 2.

Local FC safe: isImmediateMoveSafe passes but isPathSafe fails. Cost = outer scan + one FC call per move: O(N³ · N²) = O(N⁵).

Tier 3.

Desperation: counts all legal human moves after each CPU move. Triple nested loop: O(N²) CPU cells × N values × O(N³) human option count = O(N⁶).

Tier 2 worst case: O(N⁵) Tier 3 worst case: O(N⁶) Both are dominated by Tier 1 when board is not doomed.
BT Forward Check · Dominant Worst-Case
O( N² · D^N² )
D = pruned domain size after FC; D ≪ N in practice → massive real-world speedup
BT Forward Check · Theoretical Worst Case (D = N)
O( N^(N²+3) )
Only when FC prunes nothing — degenerate case, same class as Trap Setter but with higher constant
§ 04 Trap Setter — Derivation (Summary)
MRV Variable selection cost
Scan + domain size per cell: O(N²) × O(N²) = O(N⁴)
LCV Value ordering cost
LCV per variable: O(N) × O(N²) = O(N³) Combined with MRV selection: O(N²) × O(N³) = O(N⁵)
BACKTRACK Backtracking recurrence
T(d) = b · T(d − 1) → bᵈ b = N (choices per cell) d = N² (empty cells = depth) ⟹ T = N^(N²)
Trap Setter · Worst-Case (no FC propagation)
O( N^(N²) )
MRV/LCV are O(N⁵) overhead — dominated by the exponential search tree
§ 05 BT Forward Check — 3-Tier Execution Pipeline
TIER 1 · Perfect
FC passes AND full DFS path to completion is valid. Picks highest-scoring move that provably leads to a solvable board.
O(N³ · D^N²)
TIER 2 · Fallback
FC passes locally but no full safe path exists. Picks the best locally-safe move by immediate reward. Board may be doomed but move itself is legal.
O(N⁵)
TIER 3 · Desperation
Even FC fails entirely. Counts the human's remaining legal moves after each possible CPU move and picks the one that minimises human options (adversarial).
O(N⁶)
