Complexity Analysis · CSP / Towers Puzzle

Trap Setter vs. DP Strategy

A formal worst-case time complexity derivation for two solving strategies applied to the Towers puzzle, modelled as a Constraint Satisfaction Problem.

Backtracking Search MRV Heuristic LCV Heuristic Depth-Limited DP CSP
§ 01 Variable Definitions
N=board size (N × N grid)
E=number of empty cells
d=recursion depth
b=branching factor (choices per node)
k=DP lookahead depth (constant)
§ 02 Trap Setter — Step-by-Step Derivation
A MRV — Minimum Remaining Values
1.

Scan all unassigned cells to find the one with fewest remaining values. There are O(N²) cells to examine.

Scan all cells: O(N²)
2.

Computing domain size per cell checks row, column, and box constraints — each up to N values across N cells → O(N²) per cell.

Domain computation per cell: O(N²) Total MRV cost: O(N²) × O(N²) = O(N⁴)
B LCV — Least Constraining Value
1.

For each of up to N candidate values, count how many choices it eliminates from neighbouring unassigned variables.

2.

Each neighbour check: O(N) cells × O(N) constraint checks = O(N²). Combined with MRV variable-selection pass:

LCV per variable: O(N) × O(N²) = O(N³) With MRV variable selection: O(N²) × O(N³) = O(N⁵)
C Backtracking Recurrence
1.

At each search node, branch over b choices and recurse to depth d − 1:

Recurrence: T(d) = b · T(d − 1) Base case: T(0) = O(1)
2.

Unrolling the recurrence:

T(d) = b · b · T(d − 2) = b² · T(d − 2) = ... = bᵈ · T(0) = bᵈ
3.

Substituting worst-case parameters for an N × N Towers puzzle:

b = N (up to N digit choices per cell) d = N² (up to N² empty cells = max recursion depth) ⟹ T = N^(N²)
Trap Setter · Worst-Case Complexity
O( N^(N²) )
Super-exponential — intractable for large N
§ 03 Depth-Limited DP Strategy
A Depth-Limited Recurrence
1.

Same branching recurrence, but depth is capped at the compile-time constant k:

T(k) = b^k
2.

Since k is constant and independent of N, substitute b = N:

T(k) = N^k (k is fixed, not growing with N)
3.

For the typical choice of k = 3 lookahead steps:

T = N³
DP Strategy · Worst-Case (k = 3)
O( N³ )
Polynomial — tractable and scalable
